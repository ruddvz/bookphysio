Portal 4 — Admin

A

Public first, then Patient, Provider, Admin

Sequential portal-by-portal. Most logical dependency order — public site has no auth dependencies.

B

All 4 portals in parallel (separate agents)

Dispatch one agent per portal simultaneously. Faster overall delivery but needs more coordination.

Homepage + Search + Profile first (MVP core)

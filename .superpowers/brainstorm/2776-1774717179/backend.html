Backend & Data Layer

This is a full-stack clone — what powers the backend?

A

Frontend-only clone (mock data)

Build all pages with realistic mock/static data. No real database, no real auth. Perfect pixel-perfect UI demo. Fastest to build.

Pros

  • No backend complexity
  • Ships fastest
  • Pure UI focus

Cons

  • Not a working product
  • Search/booking don't actually work
B

Full-stack with Supabase

Next.js API routes + Supabase (Postgres + Auth + Storage). Real auth, real doctor listings, real bookings. Production-ready from day one.

Pros

  • Actually functional product
  • Auth, DB, storage all included
  • Scales to production

Cons

  • More complex
  • Requires Supabase setup
C

Full-stack with Prisma + PostgreSQL

Next.js API routes + Prisma ORM + PostgreSQL (local or hosted). Full control over schema and queries. More customizable than Supabase.

Pros

  • Full schema control
  • No vendor lock-in
  • Type-safe queries

Cons

  • Need to self-host or configure DB
  • Auth must be added separately (NextAuth)

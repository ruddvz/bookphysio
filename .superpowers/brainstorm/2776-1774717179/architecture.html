bookphysio.in — Full Architecture Design

Zocdoc clone · Next.js 16 + Supabase · 4 portals in parallel

Tech Stack

  • Framework: Next.js 16 (App Router, React 19)
  • UI: shadcn/ui + Tailwind CSS v4 + Lucide icons
  • Database: Supabase (PostgreSQL)
  • Auth: Supabase Auth (email, Google OAuth)
  • Storage: Supabase Storage (doctor photos, documents)
  • Search: PostgreSQL full-text search via Supabase
  • Payments: Stripe (booking deposits / provider billing)
  • Validation: Zod schemas on all inputs
  • State: Zustand (client) + React Query (server state)

Repository Structure

Project Layout

Core Database Tables

  • users — patients and providers (Supabase auth linked)
  • providers — doctor profiles, specialties, ratings
  • availabilities — provider time slots
  • appointments — bookings (patient ↔ provider ↔ slot)
  • reviews — star ratings + text
  • insurances — accepted insurance plans
  • specialties — medical specialties lookup
  • locations — clinic addresses with geocoords

4 Parallel Build Agents

Agent 1 — Public Portal

Homepage, Search Results, Doctor Profile, Specialty pages, Insurance pages, How it Works, About, Login/Signup

Agent 2 — Patient Dashboard

Dashboard home, Booking flow, Appointment history, Telehealth room, Profile/insurance settings, Notifications

Agent 3 — Provider Portal

Provider dashboard, Schedule management, Patient list, Reviews, Practice settings, Onboarding/join flow

Agent 4 — Admin Panel

Metrics dashboard, User management, Listings, Insurance/billing mgmt, Content management

Zocdoc Design System

  • Primary color: #00766C (Zocdoc teal/green)
  • Secondary: #F5F5F5 (light grey backgrounds)
  • Accent: #FF6B35 (orange CTAs)
  • Font: Inter (headings + body)
  • Border radius: 8px cards, 24px buttons
  • Doctor cards: photo + name + specialty + rating + next available slot
  • Search bar: condition/specialty + location + insurance filter

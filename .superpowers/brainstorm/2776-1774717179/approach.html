How should we approach the Zocdoc clone?

bookphysio.in — 30+ page full platform clone. Pick the build strategy.

A

Use the AI Website Cloner Template

Bootstrap from github.com/JCodesMore/ai-website-cloner-template — Next.js 16, shadcn/ui, Tailwind v4. Clone the repo, configure TARGET.md to point at Zocdoc, then build page-by-page using the 5-phase pipeline (recon → foundation → component specs → parallel build → QA). Best for pixel-perfect fidelity.

Pros

  • Proven cloning pipeline
  • Parallel agent builds
  • Modern stack (Next.js 16 + App Router)

Cons

  • Zocdoc blocks automated recon (403)
  • Must supply screenshots manually
B

Scaffold Fresh — Knowledge-Based Rebuild

Start a new Next.js 15 + shadcn/ui + Tailwind project from scratch. Build each Zocdoc page from my training knowledge of their design system. You review each section and share screenshots to correct any discrepancies. Fastest to start.

Pros

  • No template constraints
  • Cleanest architecture
  • Full control from day one

Cons

  • More setup work upfront
  • Relies on my knowledge accuracy
C

Clone Template + Custom Architecture

Use the cloner template as a skeleton (Next.js 16 + shadcn/ui already wired up), but skip the automated recon phase. Immediately jump to hand-crafting each page from Zocdoc knowledge + your screenshots. Best of both worlds.

Pros

  • Pre-built stack ready
  • Component library included
  • Skip broken recon phase

Cons

  • Template opinionated structure
  • May need to adapt conventions

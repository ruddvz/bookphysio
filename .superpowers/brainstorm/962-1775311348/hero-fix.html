Hero Section Fix — Layout Options

Current issues: (1) overflow-hidden clips animated text on some viewports, (2) min-h calc(100vh-5rem) breaks in PWA standalone mode where 100vh ≠ available height, (3) massive font (96px) wraps badly on mid-size tablets.

Current Hero — Problem Areas
⚠️ overflow-hidden on section clips the animated word when it transitions — visible flicker/cut on Safari + PWA
⚠️ min-h: calc(100vh - 5rem) — PWA standalone mode has no browser chrome so 100vh is taller than expected, making hero overflow the screen without scroll
⚠️ text-[96px] heading — on 768–1024px tablets the heading wraps mid-word, the search bar gets pushed below the fold
A

Viewport-safe + fluid type (Recommended)

Replace min-h calc(100vh-5rem) with min-h-[100svh] (small viewport height — accounts for PWA/browser chrome). Move overflow-hidden off the outer section onto only the animated word span. Scale heading with clamp(): 42px → 72px → 88px across breakpoints so it never wraps badly.

Fixes

  • PWA standalone no longer over-scrolls
  • Animated word no longer clipped
  • No mid-word wrap on tablets
  • Stats row always visible above fold

Trade-offs

  • 100svh is not supported in very old browsers (pre-2023) — negligible for India mobile market
B

Fixed height breakpoints + overflow fix only

Keep 100vh but add env(safe-area-inset-bottom) padding for PWA. Fix overflow by moving it from section to span only. Keep current font sizes but add a md:text-[72px] step to prevent tablet wrap.

Pros

  • Minimal change — fewer CSS properties touched

Cons

  • env(safe-area-inset) is unreliable in Android PWA
  • Doesn't fully solve the standalone viewport issue
C

Full responsive redesign — two-column on desktop

Redesign hero to two columns on desktop (text left, illustration/stats right), single column on mobile. Remove full-viewport-height constraint entirely — hero is naturally tall but not forced to fill screen. Animated word stays, search bar stays prominent.

Pros

  • More visual interest on desktop
  • No viewport height dependency at all
  • Search bar always above fold

Cons

  • Bigger change — risk of breaking existing design feel
  • More work than the bug fix requires

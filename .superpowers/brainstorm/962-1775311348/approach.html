Overall Implementation Approach

We have 4 areas to tackle: Hero fix · Search cards + grid · Doctor profile pipeline · Dashboard bugs+polish. How should we sequence and scope the work?

A

Sequential Polish Pass (Recommended)

One spec → one plan → implement in order: 1) Hero fix → 2) Search cards + photo → 3) Doctor profile pipeline → 4) Dashboard audit + fix. Each step is verifiable before the next begins. Cleanest git history, easiest to review.

Pros

  • Clear scope per commit
  • Easy to stop or revert mid-way
  • No dependency conflicts

Cons

  • Profile photo upload (step 3) needed before step 2 can show real photos — some wiring is deferred
B

Data-first then UI

Start with the profile photo upload pipeline (Supabase Storage + provider profile save), then surface photos everywhere simultaneously: search cards, public profile, dashboard avatars. Hero and dashboard bugs are tackled after the data layer is solid.

Pros

  • Photos are real everywhere at once
  • No placeholder-to-real migration later

Cons

  • Hero fix and dashboard bugs wait longer
  • More complex first step (backend + frontend)
C

Parallel sprints (two tracks)

Track 1: Hero fix + Search card UI upgrades (pure front-end, no backend needed, uses avatar initials as photo fallback). Track 2: Profile photo upload + provider profile enrichment + dashboard audit. Merged when both are green.

Pros

  • Fastest wall-clock time
  • UI improvements ship independently of backend

Cons

  • More complex merge
  • Search cards ship twice (once with initials, once with photos)
